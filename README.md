# **SDD Starter Pack**

### *Build software as an industry, not as a craft.*

---

## O que é o SDD Starter Pack

O **SDD Starter Pack** é um **guia prático e operacional** para implementar **Spec Driven Development (SDD)** em projetos reais — **existentes ou criados do zero**.

Ele ensina como transformar um projeto artesanal em um projeto **orientado por especificação**, onde:

* Especificações se tornam a fonte da verdade
* Código passa a ser gerado, não escrito
* Mudanças se tornam regenerações, não refatorações
* IA vira executor, humanos viram arquitetos

Este repositório é o **playbook completo de migração e operação SDD**.

---

## O problema que resolvemos

Mesmo com IA, a maioria dos projetos continua artesanal:

* Especificações implícitas
* Código difícil de evoluir
* Refatorações destrutivas
* Conhecimento preso em pessoas
* Mudanças caras e lentas

O SDD Starter Pack mostra **como sair desse modelo e operar em modo industrial.**

---

## O novo modelo

| Artesanal                   | SDD                                |
| --------------------------- | ---------------------------------- |
| Código é a fonte da verdade | Especificação é a fonte da verdade |
| Conhecimento tácito         | Conhecimento versionado            |
| Refatoração                 | Regeneração                        |
| Escala por pessoas          | Escala por processo                |
| Mudanças são arriscadas     | Mudanças são reversíveis           |

---

## O que você vai aprender

* Criar projetos 100% orientados por especificação
* Converter projetos existentes para SDD
* Criar specs executáveis
* Gerar código com IA de forma controlada
* Refatorar sem tocar no código
* Migrar stacks via especificação
* Criar pipelines de CI/CD baseados em spec

> **Note:** SDD não é apenas uma metodologia, é uma mudança de mentalidade. Você passa de "escritor de código" para "arquiteto de sistemas".

## Fluxo SDD

```
Sistema Artesanal → Especificações → Regeneração → Sistema SDD
```

---

## Primeiros passos

1. Leia `/docs/foundation/what-is-sdd.md`
2. Escolha:

   * Criar do zero → `/labs/lab-1-first-system`
   * Migrar projeto existente → `/labs/lab-2-migrate-project`
3. Crie sua primeira spec
4. Gere código
5. Itere via especificação

> **Note:** Se você está começando agora, recomendamos começar pelo Lab 1 para entender o fluxo completo antes de migrar projetos existentes.

> **Tip:** Mantenha sempre o arquivo `AGENTS.md` atualizado. Ele é essencial para garantir que a IA gere código consistente com seus padrões.

## Princípios

* Especificação antes do código
* Especificação é código
* IA executa, humano decide
* Código é derivado, não origem
* Mudanças são regenerações

> **Note:** Esses princípios são fundamentais. Violá-los significa voltar ao modelo artesanal. Sempre valide se suas práticas estão alinhadas com esses princípios.

Você não mantém mais código.
Você mantém **blueprints executáveis de sistemas.**
